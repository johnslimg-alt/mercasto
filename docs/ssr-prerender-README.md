# Пререндер главной (ветка `perf/ssr-prerender`)

Цель: убрать последнее «красное» на мобиле — **LCP ~6с**. Причина (по аудиту
PageSpeed): «самое крупное изображение недоступно прямо в HTML» — карточка
Destacados рисуется только после загрузки JS и ответа API.

Решение здесь — **build-time пререндер** (без полного SSR): после `vite build`
headless-Chromium открывает главную, даёт реальному контенту отрендериться,
и сохраняет готовый HTML обратно в `dist/index.html`. Браузер теперь рисует
реальные карточки (включая LCP-картинку) сразу из HTML.

## Почему это безопасно

- Рантайм **не меняется**: приложение по-прежнему монтируется через `createRoot()`,
  который при старте заменяет пререндеренную разметку на свежий клиентский рендер.
- Пререндер — это только «мгновенный снимок» первого экрана. Если снимок устарел
  или не собрался — приложение просто отрисует корректный контент на клиенте.
- Режим отказа: «нет улучшения», а **не** «сломанная страница». Полной гидратации
  (`hydrateRoot`), которая может падать на несовпадениях, здесь нет.

## Как проверить (на Mac, где сборка работает)

```bash
git checkout perf/ssr-prerender
npm ci                 # если нужно
npm run e2e:install    # один раз — ставит Chromium для Playwright
npm run build:prerender
```

Затем проверить результат:

```bash
# В dist/index.html внутри <div id="root"> должна появиться реальная разметка
grep -c "market-card\|Destacado" dist/index.html      # должно быть > 0
grep -c 'data-prerendered="1"' dist/index.html         # должно быть 1

# Локальный предпросмотр:
npm run preview
# открыть http://localhost:4173 и убедиться, что:
#  - первый экран рисуется мгновенно с реальными карточками
#  - нет белого экрана, тёмная/светлая тема и карта работают
#  - переходы по сайту работают как раньше
```

Прогнать Lighthouse (mobile) по `http://localhost:4173` или задеплоить ветку
на staging и сравнить LCP с текущим прод (ждём ~6с → ~2–2.5с, mobile → 85–92).

## Переменные окружения

- `PRERENDER_API_URL` — база API, к которой обращается браузер пререндера
  (по умолчанию `https://mercasto.com/api`). Должна быть доступна со сборочной
  машины, иначе карточки не отрендерятся и снимок будет без LCP-картинки.
- `PRERENDER_ROUTES` — список роутов через запятую (по умолчанию `/`).
- `PRERENDER_WAIT_MS` — таймаут ожидания контента (по умолчанию 12000).

## Если хотим убрать «мигание» снимок→клиент (опционально, после валидации)

Сейчас `createRoot` перерисовывает контент поверх снимка (картинки в кэше →
мгновенно, но возможен лёгкий flash/CLS). Чтобы убрать его полностью, можно
перейти на `hydrateRoot` в `src/main.jsx`, **но** это требует:
- одинаковой разметки снимка и клиентского рендера (иначе ошибки гидратации);
- гардов для `window`/`localStorage` (их в SSR нет — здесь не нужно, т.к.
  рендер всё равно в браузере).
Делать только после того, как базовый пререндер подтверждён на staging.

## Деплой (после успешной валидации)

В `.github/workflows/deploy-selfhosted.yml` шаг сборки фронта использует
`npm run build`. Чтобы включить пререндер в прод, заменить его на
`npm run build:prerender` (и убедиться, что на раннере установлен Chromium:
`npm run e2e:install`). **Только после** проверки на staging/локально.

## Откат

Ветка не сливается в `main` до подтверждения. Прод сейчас собирается обычным
`npm run build` и пререндера не использует — то есть ничего не меняется, пока
вы явно не переключите команду сборки.
