# Mercasto P1 Smoke Matrix\n\n## Purpose\nMinimum repeatable checks after deploy, rollback, security changes, payment changes, or infrastructure changes.\n\n## Public site checks\n- Homepage opens on HTTPS without certificate errors.\n- Homepage is not blank.\n- No public text says MVP.\n- No public text exposes internal payment provider names.\n- Search and main CTA are visible.\n- Listings page opens.\n- Listing detail page opens for active listings.\n- Paused and deleted listings are not publicly visible.\n- Images or placeholders render correctly.\n\n## Auth checks\n- Login page opens.\n- Valid user can log in.\n- Invalid credentials show safe generic error.\n- Register page opens.\n- Required fields validate.\n- Logout destroys active session.\n- Protected pages redirect guests to login.\n\n## Account checks\n- Guest cannot access account.\n- Seller sees only own listings.\n- Seller can edit own listing.\n- Seller cannot edit another user's listing.\n- Status labels render.\n- Pause, activate, delete actions are owner-only.\n\n## Photo checks\n- Owner can upload valid image.\n- Invalid extension is rejected.\n- Invalid MIME is rejected.\n- Oversized file is rejected.\n- Uploaded PHP/script files cannot execute.\n- Primary photo or placeholder renders in listing cards.\n\n## Payment checks\n- Payment page opens.\n- Public copy says payment by card or cash at OXXO where applicable.\n- Public copy does not expose internal provider names.\n- Checkout requires authenticated user.\n- Promotion purchase is only for owned listing.\n- Success/fail redirect alone does not mark payment paid.\n- Server-side confirmation is required before business effect.\n\n## Security checks\n- State-changing POST routes require CSRF where applicable.\n- Webhook routes use provider verification, not CSRF.\n- Ownership checks block access to other users' listings and photos.\n- Production does not show stack traces or raw SQL errors.\n- Logs do not contain passwords, tokens, private keys, or full session IDs.\n\n## API checks\n- /api/auth/providers returns JSON and no secrets.\n- /api/categories returns HTTP 200 and JSON array.\n- /api/ads?page=1 returns HTTP 200 and JSON pagination object.\n\n## Browser checks\n- iPhone Safari opens homepage.\n- iPhone Chrome opens homepage.\n- Desktop Chrome opens homepage.\n- No blank screen after refresh.\n\n## Server checks\n- Own PHP code passes lint.\n- git status is clean after commit/deploy.\n- No accidental secrets are committed.\n\n## P1 pass definition\nP1 passes only when public site, auth, account, photos, payments, API, and security baseline checks pass.\n
