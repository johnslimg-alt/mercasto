# Тумбнейлы изображений (ветка `feat/image-thumbnails`)

Цель: убрать рекомендацию PageSpeed «Улучшите загрузку изображений» (~478 КиБ) —
карточки на главной грузят полноразмерные оригиналы из storage.

## Что добавлено

1. **Бэкенд:** `GET /api/img?path=<storage-relative>&w=<width>`
   (`backend/app/Http/Controllers/ImageController.php`, роут в `backend/routes/api.php`).
   Отдаёт WebP-версию изображения, уменьшенную по ширине, через уже установленный
   `intervention/image` v4. Кэширует результат в `storage/app/public/cache/thumbs/`.
   - строгая валидация пути (без traversal, только под `storage/app/public`);
   - только jpg/png/webp; ширина 80..1280;
   - при любой ошибке отдаёт оригинал (картинка никогда не «ломается»).
   - nginx уже маршрутизирует `/api/...` в Laravel — правки nginx НЕ нужны.

2. **Фронт:** `src/utils/imageHelpers.js -> sizedImage()` теперь для storage-картинок
   (`/storage/....jpg|png|webp`) возвращает `/api/img?path=...&w=...`. Внешние
   (Unsplash/Picsum) — как раньше через их параметры. Детальные страницы (полный
   размер) не затронуты — `sizedImage` применяется только к карточкам.

## Как проверить на Mac

```bash
git checkout feat/image-thumbnails
# Бэкенд работает только на сервере (нужны storage + GD/Imagick), поэтому
# полноценно проверяется на сервере/staging, а не локальным preview.
```

Проверка эндпоинта (на сервере или там, где доступен backend + storage):
```
# заменить <path> на реальный путь объявления из БД, например ads/uuid.webp
curl -s -o /tmp/t.webp -w "%{content_type} %{size_download}\n" \
  "https://<staging>/api/img?path=ads/<file>.webp&w=520"
# ожидается: image/webp и размер заметно меньше оригинала
```

Фронт-проверка: открыть карточки и убедиться, что `<img src>` указывает на
`/api/img?path=...&w=...` (DevTools → Network → тип webp, размер маленький),
картинки отображаются, при недоступности эндпоинта срабатывает onError-плейсхолдер.

## Замечания по производительности

- Первый запрос каждой картинки генерирует тумбнейл (чуть медленнее), дальше —
  из кэша + `Cache-Control: immutable` на год.
- Опционально (оптимизация): прогрев кэша по cron или генерация при загрузке
  объявления, чтобы первый посетитель не ждал. Сейчас — ленивая генерация.

## Откат / безопасность

- Если `/api/img` недоступен, фронт показывает плейсхолдер (onError) — страница не
  ломается, но картинки карточек пропадут. Поэтому **проверить эндпоинт на сервере
  до слияния в `main`**.
- В `main` ничего не уходит, пока ветка не слита.

## Деплой после валидации

`/api/img` — обычный Laravel-роут, отдельной сборки не требует (бэкенд монтируется
в контейнер). После мержа в `main` достаточно стандартного деплоя. Убедиться, что в
PHP-контейнере есть расширение GD (Intervention уже используется для водяных знаков
при загрузке, значит GD доступен).
